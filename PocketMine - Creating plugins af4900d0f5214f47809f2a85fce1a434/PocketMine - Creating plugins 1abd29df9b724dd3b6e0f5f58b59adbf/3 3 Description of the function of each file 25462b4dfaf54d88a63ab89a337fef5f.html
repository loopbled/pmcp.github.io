plugin.yml and its role in plugin configuration

The plugin.yml file plays a key role in the configuration of your plugin by providing important information about it. Let's take a detailed look at its functionality.

  1. Definition of metadata:
    • name: The name of your plugin.
    • * version*: The version of your plugin.
    • main: Main class (usually specified by name_of_your_plugin\\YourPluginMain).
    • api: The minimum required version of the PocketMine-MP API.

    Example:

    yamlCopy code
name: MyFirstPlugin
version: 1.0.0
main: myfirstplugin\\MyFirstPlugin
api: 3.0.0
  1. Dependencies installation:
    • * depend*: Dependencies on other plugins.
    • softdepend: "Soft" dependencies, plugins that can be used but are not required.

    Example:

    yamlCopy code
depend:
  - AnotherPlugin
softdepend:
  - OptionalPlugin
  1. Plugin description:
    • * description*: A brief description of your plugin's functionality.
    • * author*: The author's name or nickname.
    • website: A link to the developer's website or resource.

    Example:

    yamlCopy code
description: This is my first plugin for PocketMine-MP.
author: YourName
website: <https://example.com>
  1. Other parameters:
    • load: When your plugin should be loaded (e.g. STARTUP, POSTWORLD).
    • commands: Definition of the commands provided by your plugin.

    Example:

    yamlCopy code
load: STARTUP
commands:
  mycommand:
    description: "My test command"
    usage: "/mycommand <parameter>"

The plugin.yml file provides important configuration of your plugin and is an integral part of its functionality. In the following sections, we'll look at examples in more detail and impart additional skills on how to use it.

Meaning of other files in the context of the plugin

Besides plugin.yml, other files also play an important role in the configuration and functionality of your plugin. Let's consider their significance.

  1. MyFirstPlugin.php:
    • Function: This file is the entry point to your plugin. This is where you place the main code that is executed when the plugin is activated. It may include logging events, commands, and other elements of your plugin.

    Example:

    phpCopy code
<?php

namespace myfirstplugin;

use pocketmine\\plugin\\PluginBase;

class MyFirstPlugin extends PluginBase {

    public function onEnable() {
        // Your plugin activation code
    }
}
  1. `config.yml``:
    • Function: This file stores your plugin's settings and parameters. It allows you to change the behavior of the plugin without changing the code.

    Example:

    yamlCopy code
# The settings of your plugin
setting1: value1
setting2: value2
  1. Additional files and directories:
    • Function: Depending on the specific structure of your plugin, these files may include scripts for event handling, commands, database handling, and other aspects of functionality.

    Example:

    cssCopy code
/MyFirstPlugin
└─── src
    └─── myfirstplugin
        ├─── MyFirstPlugin.php
        ├─── MyEvent.php
        └─── MyCommand.php

Each of these files contributes to the overall functionality of your plugin. Further tutorials will help you understand each of them in more depth and use them in your development.
