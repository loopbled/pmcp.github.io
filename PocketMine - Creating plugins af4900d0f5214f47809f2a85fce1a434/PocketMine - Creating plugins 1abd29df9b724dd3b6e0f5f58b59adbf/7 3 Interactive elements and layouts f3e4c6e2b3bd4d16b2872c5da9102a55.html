Adding buttons, text fields, and other elements

Interactive elements and form layouts can include different types of elements, such as buttons, text fields, and others, to allow for greater player interaction.

  1. Add Buttons:
    • Use the Forms API methods to add buttons with different properties.

    Example of adding a button with an image:

    phpCopy code
$form->addButton("Click me", 0, "path/to/image.png");
  1. Use of text fields:
    • Introducing text fields allows players to enter information directly into the form.

    Example of adding a text field:

    phpCopy code
$form->addInput("Enter your nickname");
  1. Integration of other elements:
    • In addition to buttons and text fields, forms can contain elements such as toggles, sliders, and dropdown lists.

    Example of adding a toggle:

    phpCopy code
$form->addToggle("Enable/disable option");

    Example of adding a dropdown list:

    phpCopy code
$form->addDropdown("Select an option", ["Option 1", "Option 2", "Option 3"]);

```
  1. Processing element input:
    • Add appropriate handlers for each element type to handle player actions.

    Example of button input processing:

    phpCopy code
$form = new SimpleForm(function (Player $player, int $data = null) {
    if ($data === null) {
        // Form is closed without selection
        return;
    }

    // Processing the selection from the form
    switch ($data) {
        case 0:
            // Handling button selection
            break;
        // Processing other elements
    }
});

```

Adding a variety of elements to forms enriches the user interface, providing a versatile and interesting experience of interacting with the plugin.

Adapting the interface for different devices

Flexibility and accessibility of the game plugin interface is an important aspect that ensures comfortable use on different devices. Adapting the interface for different devices allows you to create universal forms for all players.

  1. Use responsive design:
    • Use responsive UI design techniques to ensure that forms display correctly on different screen sizes.

    Example of creating an auto-scaling button:

    phpCopy code
$form->addButton("Click me", 0, "path/to/image.png", null, function (Player $player) {
    // Handling the button press
});
  1. Check for controls:
    • Periodically check that all controls remain visible and accessible on different devices.

    Example of checking the visibility of a text field:

    phpCopy code
$form->addInput("Enter your name")->setResponsive(true);
  1. Cater for differences in screen sizes:
    • Use the properties of form elements to customize their display based on screen sizes.

    Example of resizing a button:

    phpCopy code
$form->addButton("Click me", 0, "path/to/image.png")->setResponsive(true)->setSize(Button::ELEMENT_SIZE_MEDIUM);

```
  1. Testing on different devices:
    • Test the interface on different devices to make sure it displays and functions correctly.

    Example of testing on a tablet:

    phpCopy code
// Show the form on a tablet
$form->sendToPlayer($player, $player->getDevice() === Player::DEVICE_TABLET);

Adapting the interface for different devices improves the user experience and makes the plugin easier to use for players with different devices.
