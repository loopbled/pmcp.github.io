Transfer and processing of command parameters

Passing parameters in commands is an important aspect for creating functional and flexible tools on the PocketMine-MP server. Parameters allow you to dynamically adapt the behavior of a command depending on user input.

Parameter Passing

  1. Add parameters to a command:
    • Define parameters after the main command, e.g: /teleport <target>.
    • In the plugin code, parameters are passed as part of the $args array in the execute method.
  1. Example of parameter passing:
    phpCopy code
class TeleportCommand extends Command {
    public function __construct() {
        parent::__construct("teleport", "Teleport Command", "/teleport <target>");
    }

    public function execute(CommandSender $sender, string $commandLabel, array $args): bool {
        if (count($args) < 1) {
            $sender->sendMessage("Usage: /teleport <target>");
            return false;
        }

        $target = $args[0];
        // Logic of teleporting to the specified target
        // ...
        return true;
    }
}

Error handling and providing help to the user

When developing commands, it is important to anticipate possible error scenarios and provide users with the necessary help to use the command correctly. Let's look at a few key points:

Error handling for input errors

  1. Check if parameters are present:
    • Provides information about the correct use of the command in case of missing parameters.
    phpCopy code
if (count($args) < 1) {
    $sender->sendMessage("Usage: /teleport <target>");
    return false;
}
  1. Checking correctness of parameters:
    • Validation of passed data to avoid errors in further processing.
    phpCopy code
if (!is_numeric($args[0])) {
    $sender->sendMessage("Error: Parameter <target> must be a number.");
    return false;
}

