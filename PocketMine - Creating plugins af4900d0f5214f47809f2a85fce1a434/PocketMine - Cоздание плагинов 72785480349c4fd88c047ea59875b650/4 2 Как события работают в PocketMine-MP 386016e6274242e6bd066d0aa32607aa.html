В PocketMine-MP события предоставляют эффективный способ реагировать на различные действия и изменения в игре. Давайте более подробно разберем, как именно события работают в этой платформе.

Основные шаги работы с событиями

  1. Возникновение События:
    • Событие возникает в ответ на конкретное действие или изменение состояния в игре. Например, событие PlayerJoinEvent возникает, когда игрок присоединяется к серверу.
  1. Регистрация Обработчика:
    • Ваш плагин должен реализовывать интерфейс Listener. Затем вы регистрируете ваш класс-обработчик в системе событий с помощью метода registerEvents.

    Пример:

    use pocketmine\\plugin\\PluginBase;
use pocketmine\\event\\Listener;

class MyPlugin extends PluginBase implements Listener {

    public function onEnable() {
        $this->getServer()->getPluginManager()->registerEvents($this, $this);
    }
}
  1. Срабатывание Обработчика:
    • Когда событие происходит, соответствующий метод в вашем классе-обработчике вызывается автоматически. Например, если игрок присоединяется, вызывается метод onPlayerJoin.

    Пример:

    use pocketmine\\event\\player\\PlayerJoinEvent;
use pocketmine\\event\\Listener;

class MyEventListener implements Listener {

    public function onPlayerJoin(PlayerJoinEvent $event) {
        $player = $event->getPlayer();
        $player->sendMessage("Добро пожаловать, {$player->getName()}!");
    }
}
  1. Доступ к Данным События:
    • События предоставляют различные методы и свойства для доступа к данным, связанным с произошедшим событием. В приведенном выше примере мы используем метод $event->getPlayer(), чтобы получить игрока, присоединившегося к серверу.

Такой механизм обработки событий позволяет вам гибко взаимодействовать с различными аспектами игры и создавать уникальные функции для вашего сервера. В следующих частях модуля мы рассмотрим более конкретные примеры событий и их обработки.

Обзор типов событий и их возможных использований

Существует множество типов событий в PocketMine-MP, каждый из которых предоставляет уникальные возможности для взаимодействия с игровым процессом. Давайте рассмотрим несколько основных типов событий и возможные области их применения.

1. События Игроков:

  • PlayerJoinEvent: Срабатывает, когда игрок присоединяется к серверу. Используется для приветствия новых игроков, предоставления стартового оборудования и т. д.

Пример:

phpCopy code
use pocketmine\event\player\PlayerJoinEvent;
use pocketmine\event\Listener;

class MyEventListener implements Listener {

    public function onPlayerJoin(PlayerJoinEvent $event) {
        $player = $event->getPlayer();
        $player->sendMessage("Добро пожаловать, {$player->getName()}!");
    }
}
  • PlayerQuitEvent: Вызывается при выходе игрока из сервера. Может использоваться для записи статистики игроков при выходе.

2. События Блоков:

  • BlockBreakEvent: Событие разрушения блока игроком. Позволяет вам контролировать процесс добычи ресурсов.

Пример:

phpCopy code
use pocketmine\event\block\BlockBreakEvent;
use pocketmine\event\Listener;

class MyBlockListener implements Listener {

    public function onBlockBreak(BlockBreakEvent $event) {
        $player = $event->getPlayer();
        $block = $event->getBlock();

        $player->sendMessage("Вы разрушили блок {$block->getName()}");
    }
}
  • BlockPlaceEvent: Событие установки блока игроком. Позволяет вам применять дополнительные проверки перед размещением блока.

3. События Сущностей:

  • EntityDamageEvent: Вызывается при нанесении урона сущности. Может использоваться для модификации урона или предотвращения повреждений.

Пример:

phpCopy code
use pocketmine\event\entity\EntityDamageEvent;
use pocketmine\event\Listener;

class MyEntityListener implements Listener {

    public function onEntityDamage(EntityDamageEvent $event) {
        $entity = $event->getEntity();
        $damage = $event->getFinalDamage();

        if ($entity instanceof Player) {
            $entity->sendMessage("Вы получили урон: $damage");
        }
    }
}

Это лишь небольшой обзор типов событий и их возможных использований. В зависимости от ваших потребностей вы можете использовать другие события и настраивать их обработчики для создания уникального игрового опыта на вашем сервере.

Примеры популярных событий

В этом разделе мы рассмотрим несколько примеров популярных событий и их обработки в PocketMine-MP.

1. Событие присоединения игрока:

phpCopy code
use pocketmine\event\player\PlayerJoinEvent;
use pocketmine\event\Listener;

class MyEventListener implements Listener {

    public function onPlayerJoin(PlayerJoinEvent $event) {
        $player = $event->getPlayer();
        $player->sendMessage("Добро пожаловать, {$player->getName()}!");
    }
}

Этот пример показывает, как обрабатывать событие присоединения игрока. При каждом присоединении игрока к серверу отправляется приветственное сообщение.

2. Событие разрушения блока:

phpCopy code
use pocketmine\event\block\BlockBreakEvent;
use pocketmine\event\Listener;

class MyBlockListener implements Listener {

    public function onBlockBreak(BlockBreakEvent $event) {
        $player = $event->getPlayer();
        $block = $event->getBlock();

        $player->sendMessage("Вы разрушили блок {$block->getName()}");
    }
}

Этот пример показывает, как обрабатывать событие разрушения блока игроком. При каждом разрушении блока отправляется сообщение с информацией о разрушенном блоке.

3. Событие получения урона сущностью:

phpCopy code
use pocketmine\event\entity\EntityDamageEvent;
use pocketmine\event\Listener;

class MyEntityListener implements Listener {

    public function onEntityDamage(EntityDamageEvent $event) {
        $entity = $event->getEntity();
        $damage = $event->getFinalDamage();

        if ($entity instanceof Player) {
            $entity->sendMessage("Вы получили урон: $damage");
        }
    }
}

Этот пример показывает, как обрабатывать событие получения урона сущностью (например, игроком). При каждом получении урона отправляется сообщение с информацией о полученном уроне.

Это всего лишь несколько примеров того, как события используются для взаимодействия с различными аспектами игры. В зависимости от ваших потребностей вы можете создавать свои собственные обработчики для других событий и дополнять функциональность вашего сервера.
