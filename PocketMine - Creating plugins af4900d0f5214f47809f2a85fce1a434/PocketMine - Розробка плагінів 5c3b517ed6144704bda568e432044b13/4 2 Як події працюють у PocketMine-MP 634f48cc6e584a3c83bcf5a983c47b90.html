У PocketMine-MP події надають ефективний спосіб реагувати на різні дії та зміни в грі. Давайте детальніше розберемо, як саме події працюють у цій платформі.

Основні кроки роботи з подіями.

  1. Виникнення Події:
    • Подія виникає у відповідь на конкретну дію або зміну стану в грі. Наприклад, подія PlayerJoinEvent виникає, коли гравець приєднується до сервера.
  1. Реєстрація Обробника:
    • Ваш плагін повинен реалізовувати інтерфейс Listener. Потім ви реєструєте ваш клас-обробник у системі подій за допомогою методу registerEvents.

    Приклад:

    use pocketmine\\\\plugin\\\\PluginBase;
use pocketmine\\\\event\\\\Listener;

class MyPlugin extends PluginBase implements Listener {

    public function onEnable() {
        $this->getServer()->getPluginManager()->registerEvents($this, $this);
    }
}
  1. Спрацьовування Обробника:
    • Коли подія відбувається, відповідний метод у вашому класі-обробнику викликається автоматично. Наприклад, якщо гравець приєднується, викликається метод onPlayerJoin.

    Приклад:

    use pocketmine\\\\event\\\\player\\\\PlayerJoinEvent;
use pocketmine\\\\event\\\\Listener;

class MyEventListener implements Listener {

    public function onPlayerJoin(PlayerJoinEvent $event) {
        $player = $event->getPlayer();
        $player->sendMessage("Ласкаво просимо, {$player->getName()}!");
    }
}
  1. Доступ до Даних Події:
    • Події надають різні методи і властивості для доступу до даних, пов'язаних із подією, що відбулася. У наведеному вище прикладі ми використовуємо метод $event->getPlayer(), щоб отримати гравця, який приєднався до сервера.

Такий механізм обробки подій дає змогу вам гнучко взаємодіяти з різними аспектами гри та створювати унікальні функції для вашого сервера. У наступних частинах модуля ми розглянемо більш конкретні приклади подій та їхньої обробки.

Огляд типів подій та їхніх можливих використань

Існує безліч типів подій у PocketMine-MP, кожен з яких надає унікальні можливості для взаємодії з ігровим процесом. Розгляньмо кілька основних типів подій і можливі сфери їх застосування.

1. Події Гравців:

  • PlayerJoinEvent: Спрацьовує, коли гравець приєднується до сервера. Використовується для привітання нових гравців, надання стартового обладнання тощо.

Приклад:

phpCopy code
use pocketmine\\event\\player\\PlayerJoinEvent;
use pocketmine\\event\\Listener;

class MyEventListener implements Listener {

    public function onPlayerJoin(PlayerJoinEvent $event) {
        $player = $event->getPlayer();
        $player->sendMessage("Ласкаво просимо, {$player->getName()}!");
    }
}
  • PlayerQuitEvent: Викликається при виході гравця з сервера. Може використовуватися для запису статистики гравців при виході.

2. Події Блоків:

  • BlockBreakEvent: Подія руйнування блоку гравцем. Дозволяє вам контролювати процес видобутку ресурсів.

Приклад:

phpCopy code
use pocketmine\\event\\block\\BlockBreakEvent;
use pocketmine\\event\\Listener;

class MyBlockListener implements Listener {

    public function onBlockBreak(BlockBreakEvent $event) {
        $player = $event->getPlayer();
        $block = $event->getBlock();

        $player->sendMessage("Ви зруйнували блок {$block->getName()}");
    }
}
  • BlockPlaceEvent: Подія встановлення блоку гравцем. Дозволяє вам застосовувати додаткові перевірки перед розміщенням блоку.

3. Події Сутностей:

  • EntityDamageEvent: Викликається при нанесенні шкоди сутності. Може використовуватися для модифікації шкоди або запобігання пошкоджень.

Приклад:

phpCopy code
use pocketmine\\event\\entity\\EntityDamageEvent;
use pocketmine\\event\\Listener;

class MyEntityListener implements Listener {

    public function onEntityDamage(EntityDamageEvent $event) {
        $entity = $event->getEntity();
        $damage = $event->getFinalDamage();

        if ($entity instanceof Player) {
            $entity->sendMessage("Ви отримали шкоду: $damage");
        }
    }
}

Це лише невеликий огляд типів подій та їхніх можливих використань. Залежно від ваших потреб ви можете використовувати інші події та налаштовувати їх обробники для створення унікального ігрового досвіду на вашому сервері.

Приклади популярних подій

У цьому розділі ми розглянемо кілька прикладів популярних подій та їхньої обробки в PocketMine-MP.

1. Подія приєднання гравця:

phpCopy code
use pocketmine\\event\\player\\PlayerJoinEvent;
use pocketmine\\event\\Listener;


class MyEventListener implements Listener {


    public function onPlayerJoin(PlayerJoinEvent $event) {
        $player = $event->getPlayer();
        $player->sendMessage("Ласкаво просимо, {$player->getName()}!");
    }
}

Цей приклад показує, як обробляти подію приєднання гравця. При кожному приєднанні гравця до сервера надсилається вітальне повідомлення.

2. Подія руйнування блоку:

phpCopy code
use pocketmine\\event\\block\\BlockBreakEvent;
use pocketmine\\event\\Listener;


class MyBlockListener implements Listener {


    public function onBlockBreak(BlockBreakEvent $event) {
        $player = $event->getPlayer();
        $block = $event->getBlock();


        $player->sendMessage("Ви зруйнували блок {$block->getName()}");
    }
}

Цей приклад показує, як обробляти подію руйнування блоку гравцем. При кожному руйнуванні блоку надсилається повідомлення з інформацією про зруйнований блок.

3. Подія отримання шкоди сутністю:

phpCopy code
use pocketmine\\event\\entity\\EntityDamageEvent;
use pocketmine\\event\\Listener;


class MyEntityListener implements Listener {


    public function onEntityDamage(EntityDamageEvent $event) {
        $entity = $event->getEntity();
        $damage = $event->getFinalDamage();


        if ($entity instanceof Player) {
            $entity->sendMessage("Ви отримали шкоду: $damage");
        }
    }
}

Цей приклад показує, як обробляти подію отримання шкоди сутністю (наприклад, гравцем). При кожному отриманні шкоди надсилається повідомлення з інформацією про отриману шкоду.

Це всього лише кілька прикладів того, як події використовуються для взаємодії з різними аспектами гри. Залежно від ваших потреб ви можете створювати свої власні обробники для інших подій і доповнювати функціональність вашого сервера.
