Передача та обробка параметрів команд

Передача параметрів у командах є важливим аспектом для створення функціональних і гнучких інструментів на сервері PocketMine-MP. Параметри дають змогу динамічно адаптувати поведінку команди залежно від введення користувача.

Передача параметрів

  1. Добавлення параметрів у команду:
    • Визначення параметрів після основної команди, наприклад: **/teleport <мета>***.
    • У коді плагіна параметри передаються як частина масиву $args у методі execute.
  1. Приклад передачі параметрів:
    phpCopy code
class TeleportCommand extends Command {
    public function __construct() {
        parent::__construct("teleport", "Команда для телепортації", "/teleport <ціль>");
    }

    public function execute(CommandSender $sender, string $commandLabel, array $args): bool {
        if (count($args) < 1) {
            $sender->sendMessage("Використання: /teleport <ціль>");
            return false;
        }

        $target = $args[0];
        // Логіка телепортації до вказаної цілі
        // ...
        return true;
    }
}

Обробка параметрів.

  1. Перевірка наявності параметрів:
    • Перевірка кількості переданих параметрів.
    • Надання довідки в разі відсутності параметрів.
  1. Обробка отриманих параметрів:
    • Використання переданих параметрів у логіці команди.
    • Перевірка типів даних і валідація значень.

Приклад обробки параметрів:

public function execute(CommandSender $sender, string $commandLabel, array $args): bool {

    // ...

Обробка помилок і надання довідки користувачеві

Під час розроблення команд важливо передбачити сценарії можливих помилок і забезпечити користувачів необхідною довідкою для правильного використання команди. Розглянемо кілька ключових моментів:

Обробка помилок введення.

  1. Перевірка наявності параметрів:
    • Надання інформації про коректне використання команди в разі відсутності параметрів.
    phpCopy code
if (count($args) < 1) {
    $sender->sendMessage("Використання: /teleport <ціль>");
    return false;
}
  1. Перевірка коректності параметрів:
    • Валідація переданих даних, щоб уникнути помилок у подальшій обробці.
    phpCopy код
if (!is_numeric($args[0])) {
    $sender->sendMessage("Помилка: Параметр <ціль> повинен бути числом.");
    return false;
}

Надання довідки.

  1. Повідкове повідомлення при помилці:
    • Надання інформації про правильний формат використання команди.
    phpCopy code
$sender->sendMessage("Використання: /teleport <ціль>");
  1. Подробна довідка по команді:
    • Додаткова інформація про можливі параметри та їхнє використання.
    phpCopy code
public function __construct() {
    parent::__construct("teleport", "Команда для телепортації", "/teleport <ціль>");
}

Приклад команди з обробкою помилок і довідкою:

phpCopy code
class TeleportCommand extends Command {
    public function __construct() {
        parent::__construct("teleport", "Команда для телепортації", "/teleport <ціль>");
    }


    public function execute(CommandSender $sender, string $commandLabel, array $args): bool {
        if (count($args) < 1 || !is_numeric($args[0])) {
            $sender->sendMessage("Використання: /teleport <ціль>");
            return false;
        }


        $target = $args[0];
        // Логіка телепортації до вказаної цілі
        // ...


        return true;
    }
}

Ефективна обробка помилок і надання зрозумілої довідки робить використання ваших команд приємним та інтуїтивним для гравців.
